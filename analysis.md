# 13 · 코드잇 영상 댓글 분석 웹자료 구성안

> 목적: 코드잇 영상 4개의 댓글 캡처 원문을 바탕으로, 시청자 반응을 콘텐츠 기획 신호와 HTML/CSS/JSON 웹자료 구조로 정리한다.

## 1. 분석 목적

코드잇 과제의 핵심은 전환 유도가 아니라 인지다. 즉 "코드잇은 강의 퀄리티, 학습 경험, 교육 진정성에 집착하는 회사"라는 인식을 유튜브 콘텐츠로 남겨야 한다. 댓글 분석은 이 메시지를 누구에게 어떤 질문으로 꺼낼지 정하는 근거다. 다만 강의 퀄리티 자체는 현재 댓글에서 직접 증명된 결론이 아니라, 다음 콘텐츠가 보여줘야 할 목표로 분리한다.

이번 웹자료는 댓글을 단순 반응 모음으로 보지 않고 다음 세 가지로 재해석한다.

| 관점 | 질문 | 산출물 반영 |
|---|---|---|
| 시청자 심리 | 이 사람은 왜 댓글을 남겼는가 | 페르소나와 intent 필드 |
| 콘텐츠 신호 | 어떤 장면이 반응을 만들었는가 | 댓글 유형, 키워드, 영상별 main_signal |
| 기획 전환 | 다음 영상에서 무엇을 증명해야 하는가 | content_implication, 기획 제안 |

### 근거 수준 구분

| 구분 | 의미 | 이 자료의 처리 |
|---|---|---|
| 댓글 직접 신호 | 댓글 요약 또는 짧은 원문 인용에서 확인되는 반응 | 공간 호감, 후속 요청, 개발자 관심, AI 취업 불안 |
| 기획 해석 | 댓글 신호를 과제 목표에 맞게 다음 콘텐츠 구조로 번역한 판단 | 강의 퀄리티 집착, 안티마케팅 톤, 제작 비하인드 장면 |
| 캡처 기준 신호 | 확인 가능한 댓글 캡처에서 읽히는 반응 | AI 불안, 후속 요청, 지방 학습 관심, 위워크 학습 경험 |

## 3. 댓글 유형과 페르소나

### 댓글 4대 유형

| 유형 | 해석 | 기획 활용 |
|---|---|---|
| 공간·편의시설 칭찬 | 학습 공간, 간식, 모니터, 루틴이 진입 호감을 만든다 | 초반 후킹으로 쓰되 공간 자랑에서 끝내지 않는다 |
| 후속 시리즈 요청 | 인물과 루틴에 정이 붙은 시청자가 다음 이야기를 원한다 | 시즌제, 다음 회차 예고, 댓글 질문 회수 구조가 필요하다 |
| 개발자 관심도 증가 | 실무 학습법, 질문 문화, 채용공고 독해에 반응한다 | 기술을 쉽게 번역하는 설명형 세그먼트가 필요하다 |
| AI 시대 개발자 직업 전망 불안 | AI로 인한 채용 축소와 부트캠프 ROI를 걱정한다 | 반복 확인된 질문으로 다루되, 공포 조장이 아니라 학습 설계와 역량 증명으로 답한다 |

### 우선 페르소나와 보조 신호

| 페르소나 | 대표 불안 또는 욕구 | 다음 콘텐츠 역할 |
|---|---|---|
| P1. AI 취업 불안 회의론자 | AI가 주니어 채용을 줄이면 지금 시작해도 되는가 | 시리즈의 도입 질문 |
| P2. 막막한 예비 수강생 | 비전공, 지방, 늦은 시작도 가능한가 | 학습자 서사의 주인공 |
| P3. 공간·분위기 동경형 | 저런 환경에서 공부하면 나도 달라질 것 같다 | 진입 호감과 공유 동기 |
| P5. 응원·공감 코어팬 | 이 사람의 다음 이야기를 보고 싶다 | 연재와 댓글 커뮤니티의 씨앗 |
| P4. 크리에이터 지향 개발자 동료 | 학습 설계와 제작 방식을 참고하고 싶다 | 보조 확산 신호 |
| P6. 게임·AI 실험 라이프스타일러 | 개발, 게임, AI, 라이프스타일을 함께 소비한다 | 보조 확장 신호 |
| P7. 휴먼중심 실무 사고자 | 기술은 결국 사람의 문제를 해결해야 한다 | 댓글 1개 기반의 보조 실무 언어 신호 |

## 4. 참고 웹사이트 구조 분석

참고한 댓글 분석형 웹자료는 단일 페이지 안에서 데이터 파일을 읽어 통계, 키워드, 테마, 아카이브를 렌더링하는 구조였다. 코드잇 웹자료도 같은 방향으로 HTML, CSS, JSON, JavaScript를 분리해 재사용 가능한 형태로 구성한다.

확인된 구조는 다음과 같다.

| 영역 | 구성 | 코드잇 적용 |
|---|---|---|
| 히어로 | 제목, 분석 대상, 주요 지표 | "응원하고 싶다 / 그래서 나는 될까"라는 댓글 핵심 정서로 구성 |
| 요약 지표 | 선별 댓글 수, 공감 수, 긴 사연 등 | 분석 영상 수, 댓글 수, 유형 수, 연결 페르소나 수 |
| 테마 분류 | 감정 또는 테마별 댓글 분포 | 댓글 4대 유형 분포 |
| 키워드 클라우드 | 반복 단어 집계 | AI, 취업, 비전공, 지방, 질문, 동료 등 |
| 지도 또는 맵 | 지역 댓글을 GeoJSON으로 표현 | 코드잇은 지역 지도 대신 페르소나 맵과 영상별 테이블 사용 |
| 댓글 아카이브 | 원문, 좋아요, 테마, 정렬 | 댓글 원문 카드, 영상 필터, 유형 필터, 정렬 |

예시 사이트는 CSV와 지도 데이터를 함께 사용하지만, 코드잇 자료는 구조화된 JSON이 더 적합하다. 이유는 영상, 댓글, 테마의 관계가 명확하고, 같은 화면을 다른 영상 분석에도 쉽게 재사용할 수 있기 때문이다.

## 6. HTML/CSS 화면 설계

구현 위치: `_workspace/codeit-comment-web/index.html`

화면 순서는 다음과 같다.

1. 개요 히어로: 댓글의 핵심 정서를 한 문장으로 제시한다.
2. 분석한 영상: 영상 4개 썸네일을 배치하고, 클릭하면 유튜브 원본으로 이동한다.
3. 댓글 요약 지표: 영상 수, 댓글 수, 유형 수, 페르소나 수를 노출한다.
4. 댓글 신호: 유형별 막대 그래프와 해석을 보여준다.
5. 키워드와 페르소나: 키워드 클라우드와 페르소나 분포를 보여준다.
6. 영상별 분석: 영상별 주요 신호와 기획 함의를 표로 정리한다.
7. 댓글 원문 아카이브: 영상 필터, 유형 필터, 정렬을 제공한다.
8. 콘텐츠 기획 제안: AI 불안 질문, 공간에서 학습 경험으로의 이동, 시즌 구조를 제안한다.

### 디자인 원칙

| 기준 | 적용 |
|---|---|
| Codeit Violet | `#6500C3`을 강조와 선택 상태에만 사용 |
| 과장 없는 톤 | 광고식 카피보다 댓글 근거와 기획 해석 중심 |
| 보편적 구조 | JSON만 교체하면 다른 영상 댓글에도 재사용 가능 |
| 몰입 | 카드와 그래프는 간결하게, 원문과 해석의 관계를 우선 |
| 접근성 | 필터에 aria label, 링크에 명확한 제목, 반응형 레이아웃 적용 |
| 보안 | 유튜브 링크와 hex 색상만 허용해 JSON 교체 시 링크·스타일 삽입 위험을 낮춤 |

## 7. 실행 방법

웹자료는 두 방식으로 확인할 수 있다. 파일을 직접 열면 `data-inline.js`의 내장 데이터를 사용하고, 정적 서버로 실행하면 `data/*.json` 파일을 읽는다.

파일 직접 열기:

```text
_workspace/codeit-comment-web/index.html
```

정적 서버 실행:

```bash
cd /Users/parkjackson/Documents/코드잇\ 사전과제/_workspace/codeit-comment-web
python3 -m http.server 4173
```

브라우저에서 다음 주소를 연다.

```text
http://127.0.0.1:4173/
```

## 9. 현재 한계와 다음 단계

현재 웹자료는 캡처에서 확인한 댓글 10개를 우선 반영했다. 추가 댓글 캡처나 CSV가 생기면 `comments.json`과 `data-inline.js`를 같은 내용으로 갱신하면 된다.

다음 단계는 다음과 같다.

1. 추가 댓글 캡처 또는 CSV를 확보한다.
2. 원핸들은 화면에 노출하지 않고 `author_alias`를 댓글 번호로 유지한다.
3. 댓글별 `theme`, `persona`, `intent`를 재분류한다.
4. 같은 HTML 구조에서 데이터만 교체해 웹자료를 갱신한다.
